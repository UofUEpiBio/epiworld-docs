# Epiworld Documentation

[![MkDocs Documentation](https://github.com/UofUEpiBio/epiworld-docs/actions/workflows/doxy-action.yml/badge.svg)](https://github.com/UofUEpiBio/epiworld-docs/actions/workflows/doxy-action.yml)

This repository hosts the documentation website for the
[**epiworld**](https://github.com/UofUEpiBio/epiworld) C++ library, a
general-purpose framework for agent-based epidemiological simulation.

**Live site:** <https://UofUEpiBio.github.io/epiworld-docs>

## Repository Structure

```text
epiworld-docs/
├── docs_src/
│   ├── README.md          # Homepage
│   ├── guide/             # Narrative learning path
│   ├── examples/          # Example gallery and synced upstream examples
│   ├── reference/         # C++ API landing page
│   ├── concepts/          # Legacy concept pages kept for cross-links
│   ├── impl/              # Legacy implementation notes kept for cross-links
│   ├── models/            # Hand-authored model notes kept for cross-links
│   ├── assets/            # Images and branding
│   └── javascripts/       # MathJax configuration
├── scripts/
│   └── sync_examples.py   # Generates docs pages from the main epiworld repo
├── mkdocs.yml             # MkDocs configuration
└── .github/workflows/     # CI/CD pipeline
```

## Documentation Structure

The site is organized around three reader-facing sections:

- `Guide`: conceptual chapters, model-building tutorials, and built-in model
  overviews.
- `Examples`: a browsable gallery of code pulled from the upstream
  [epiworld](https://github.com/UofUEpiBio/epiworld) repository.
- `C++ API`: the generated Doxygen reference for classes, files, and
  namespaces.

This keeps the docs aligned with three common user tasks: learning epiworld,
finding runnable examples, and consulting the API.

## How The Site Is Built

The documentation website is built with [MkDocs](https://www.mkdocs.org/) and
the [Material for MkDocs](https://squidfunk.github.io/mkdocs-material/) theme.
The API reference is generated from the epiworld source code using
[mkdoxy](https://github.com/JakubAndrysek/MkDoxy), and the example gallery is
generated from the upstream repository with `scripts/sync_examples.py`.

This repository does **not** vendor the epiworld source code. During CI, the
workflow clones the upstream repository into `epiworld-src/`, syncs example
pages, builds the Doxygen-backed API reference, and then builds the MkDocs
site.

## Local Development

To build the documentation locally:

```bash
# Clone the epiworld source for API docs and synced examples
git clone --depth 1 https://github.com/UofUEpiBio/epiworld.git epiworld-src

# Install dependencies
pip install mkdocs mkdocs-material mkdoxy
sudo apt-get install -y doxygen graphviz  # or use brew on macOS

# Sync example pages from the upstream repository
python3 scripts/sync_examples.py --source epiworld-src

# Build and serve locally
mkdocs serve
```

Then visit <http://127.0.0.1:8000> in your browser.

## Authoring Notes

- New AI-generated chapters should begin with the warning:
  `This chapter was automatically generated by an AI agent and has not been reviewed by a human. Read with caution`.
- Prefer adding user-facing narrative content under `docs_src/guide/`.
- Prefer keeping detailed API material in the upstream source comments so it
  flows into the generated C++ API reference.
- Prefer adding runnable example code to the main `epiworld` repository, then
  letting `scripts/sync_examples.py` expose it on the docs site.
