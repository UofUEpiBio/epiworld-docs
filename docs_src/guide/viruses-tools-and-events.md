## C++ Pattern

The existing implementation notes show the basic event-driven workflow:

```cpp
template<typename TSeq>
void lockdown_event(epiworld::Model<TSeq>* model) {
    double tr = model->get_param("Transmission rate");
    model->set_param("Transmission rate", tr * 0.5);
}

template<typename TSeq>
void new_variant_event(epiworld::Model<TSeq>* model) {
    epiworld::Virus<TSeq> variant("Variant B", 0.01, true);
    variant.set_prob_infecting("Transmission rate");
    model->add_virus(variant);
}

model.add_globalevent(lockdown_event<int>, "Lockdown Policy", 30);
model.add_globalevent(new_variant_event<int>, "Variant Introduction", 45);
```

## R Pattern

The R interface already documents a straightforward virus-augmentation workflow
for built-in models:

```r
library(epiworldR)

model <- ModelSEIRCONN(
  name = "COVID-19",
  prevalence = 0.01,
  n = 10000,
  contact_rate = 10,
  incubation_days = 7,
  transmission_rate = 0.10,
  recovery_rate = 1 / 7
) |> add_virus(
  virus(
    name = "COVID-19 (delta)",
    prevalence = 0.01,
    as_proportion = TRUE,
    prob_infecting = 0.20,
    recovery_rate = 0.60,
    prob_death = 0.50,
    incubation = 7
  )
)

run(model, ndays = 100)
```
