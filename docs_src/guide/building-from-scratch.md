The custom-model workflow in epiworld centers on `Model<TSeq>::add_state()`.
Each state gets an update function, and that function decides what agents in
that state do during a simulation step.

## Design Flow

1. List the states you need.
2. Write one update function per state.
3. Register the states with `add_state()`.
4. Initialize the population and contact structure.
5. Add any viruses, tools, and global events.
6. Run the model and inspect the recorded outputs.

## Core Pattern

```cpp
template<typename TSeq>
void update_susceptible(epiworld::Agent<TSeq>* agent, epiworld::Model<TSeq>* model) {
    // Custom exposure logic goes here.
}

template<typename TSeq>
void update_infected(epiworld::Agent<TSeq>* agent, epiworld::Model<TSeq>* model) {
    // Custom recovery or progression logic goes here.
}

template<typename TSeq>
void update_recovered(epiworld::Agent<TSeq>* agent, epiworld::Model<TSeq>* model) {
    // Custom immunity logic goes here.
}

template<typename TSeq>
void update_isolated(epiworld::Agent<TSeq>* agent, epiworld::Model<TSeq>* model) {
    if (model->runif() < 0.05)
        agent->change_status(model->get_state_id("Susceptible"));
}

epiworld::Model<TSeq> model;
model.add_state("Susceptible", update_susceptible<TSeq>);
model.add_state("Infected", update_infected<TSeq>);
model.add_state("Recovered", update_recovered<TSeq>);
model.add_state("Isolated", update_isolated<TSeq>);
```

## What `add_state()` Gives You

- Full control over agent-level transitions.
- A direct path to non-standard compartments and behaviors.
- A clean way to mix built-in and user-defined logic in the same model.

## R and Cross-Interface Workflows

The current R interface is strongest when you want to run built-in models,
compare parameter sets, or analyze output. The custom `add_state()` workflow is
best documented from the C++ side, then paired with R examples for downstream
analysis or built-in-model prototyping.

```r
library(epiworldR)

sir <- ModelSIR(
  name = "Prototype",
  prevalence = 0.01,
  transmission_rate = 0.20,
  recovery = 0.10
) |>
  agents_smallworld(n = 5000, k = 8, d = FALSE, p = 0.01) |>
  run(ndays = 60, seed = 42)

plot(sir)
```
