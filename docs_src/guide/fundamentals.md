The core idea behind epiworld is simple: a model is a population of agents
whose states change over time as they interact through a network or mixing
structure.

## The Main Building Blocks

| Concept | Why it matters | Related detail |

## A Minimal Built-in Model

=== "C++"

    ```cpp
    #include "epiworld.hpp"

    using namespace epiworld;

    int main() {
        epimodels::ModelSIR<> model(
            "COVID-19",
            0.01,
            0.10,
            0.30
        );

        model.agents_smallworld(100000, 10, false, 0.01);
        model.run(100, 122);
        model.print();
    }
    ```

=== "R"

    ```r
    library(epiworldR)

    sir <- ModelSIR(
      name = "COVID-19",
      prevalence = 0.01,
      transmission_rate = 0.10,
      recovery = 0.30
    ) |>
      agents_smallworld(n = 100000, k = 10, d = FALSE, p = 0.01) |>
      run(ndays = 100, seed = 122)

    summary(sir)
    ```

## A Good Mental Model

- Start with states and the rules for moving between them.
- Decide whether contacts come from a network, a fully connected population, or
  a mixing matrix.
- Add pathogens, interventions, and events only after the transition logic is
  clear.
- Treat the built-in models as templates you can run directly or use as
  examples for custom work.
