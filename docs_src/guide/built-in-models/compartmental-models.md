This family covers the standard compartmental templates that run on explicit
contact networks.

## Models In This Family

| Model | Description | API |
| --- | --- | --- |
| SIS | Reinfection is possible. | [classModelSIS](../../epiworld/classModelSIS.md) |
| SIR | Recovery produces a removed or immune state. | [classModelSIR](../../epiworld/classModelSIR.md) |
| SEIR | Adds an exposed state before infectiousness. | [classModelSEIR](../../epiworld/classModelSEIR.md) |
| SIRD | Adds a death compartment to SIR. | [classModelSIRD](../../epiworld/classModelSIRD.md) |
| SISD | Adds a death compartment to SIS. | [classModelSISD](../../epiworld/classModelSISD.md) |
| SEIRD | Adds a death compartment to SEIR. | [classModelSEIRD](../../epiworld/classModelSEIRD.md) |

## Minimal Example

For this family, the constructor signature varies with the compartments you
choose. The `ModelSIR` pattern is the simplest baseline:

=== "C++"

    ```cpp
    epimodels::ModelSIR<> model("Influenza", 0.01, 0.15, 0.20);
    model.agents_smallworld(10000, 8, false, 0.01);
    model.run(90, 321);
    ```

=== "R"

    ```r
    library(epiworldR)

    model <- ModelSEIR(
      name = "Influenza",
      prevalence = 0.01,
      transmission_rate = 0.15,
      recovery_rate = 0.20,
      incubation_days = 10
    ) |>
      agents_smallworld(n = 10000, k = 8, d = FALSE, p = 0.01) |>
      run(ndays = 90, seed = 321)
    ```

Use the linked API pages above for the exact argument list of each variant.

## Choose This Family When

- You want a network-based epidemic model with familiar compartments.
- You do not need mixing matrices or disease-specific quarantine logic.
- You want a clean baseline before adding custom states or interventions.
