# Common Patterns

Most built-in epiworld models differ in compartments and contact structure, but
they still follow the same broad usage pattern.

## Shared Workflow

1. Construct a model with epidemiological parameters.
2. Define the population structure, such as a network or connected population.
3. Optionally add viruses, tools, entities, or global events.
4. Run the model.
5. Summarize or plot the results.

## Shared Questions To Answer

| Question | Why it matters |
| --- | --- |
| Are contacts network-based, fully connected, or matrix-based? | This determines which model family fits best. |
| Does the model include an exposed or death compartment? | This changes the state machine and parameters you must provide. |
| Are interventions part of the scenario? | If yes, avoid rewriting the same virus/tool/event explanation in every page. |
| Do you need disease-specific behavior? | If yes, use a specialized model or extend one from C++. |

## Reusable Example Pattern

=== "C++"

    ```cpp
    epimodels::ModelSIR<> model("COVID-19", 0.01, 0.10, 0.30);
    model.agents_smallworld(10000, 8, false, 0.01);
    model.run(60, 123);
    model.print();
    ```

=== "R"

    ```r
    library(epiworldR)

    sir <- ModelSIR(
      name = "COVID-19",
      prevalence = 0.01,
      transmission_rate = 0.10,
      recovery = 0.30
    ) |>
      agents_smallworld(n = 10000, k = 8, d = FALSE, p = 0.01) |>
      run(ndays = 60, seed = 123)
    ```
