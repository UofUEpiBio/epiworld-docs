# Mixing Models

Mixing models are the right choice when different groups interact at different
rates and a contact matrix is part of the scientific question.

## Minimal Example

These models all require a contact matrix. The exact constructor signature
varies by class, so use the linked API pages above for the final argument list.

=== "C++"

    ```cpp
    std::vector<double> contact_matrix = {
        0.80, 0.20,
        0.30, 0.70
    };

    epimodels::ModelSEIRMixing<> model(
        /* family-specific arguments */,
        contact_matrix
    );
    ```

=== "R"

    ```r
    library(epiworldR)

    contact_matrix <- c(
      0.80, 0.20,
      0.30, 0.70
    )

    model <- ModelSEIRMixing(
      name = "Influenza",
      n = 1000,
      prevalence = 0.01,
      contact_rate = 4,
      transmission_rate = 0.15,
      incubation_days = 7,
      recovery_rate = 1 / 5,
      mixing = contact_matrix
    )
    ```

## Choose This Family When

- Population structure matters more than explicit edge-by-edge networks.
- You need entities, groups, or contact matrices in the core model definition.
- You want to explain matrix logic once and reuse it across several model pages.
